## 1. 市场规模（TAM / SAM / SOM）

### 1.1 总盘子（TAM）
- **中国网络零售**：年成交额量级在 **15 万亿+ 人民币**（淘天 / 京东 / 拼多多 / 抖音电商合计），全球最大单一电商市场。
- **美国电商**：年成交额量级在 **1.2 万亿+ 美元**，且增速稳定、客单价高、利润空间大。
- 两国合计覆盖全球电商 GMV 的大半，是任何全球级购物产品的必争之地。

### 1.2 Zeno 切入的细分（SAM）
Zeno 不直接做 GMV，而是做 **"AI 引导的成交"**——对应：
- **导购 / 联盟 / 返利市场**：中国有成熟的淘客 / 返利 / 什么值得买生态（年引导 GMV 万亿量级）；美国有 Rakuten / Honey / Wirecutter / RetailMeNot 等。
- **AI 购物导流**：2025 假期 AI 导流增长约 700% 且高转化，是一个**正在从零爆发**的新市场。
- Zeno 的 SAM = "可被 AI 重新中介化的导购 + 新增 AI 导流"，是一个**存量替代 + 增量爆发**叠加的市场。

### 1.3 可获取（SOM，前 3 年，假设）
- 聚焦 **2–4 个高决策成本品类**（见 §5）起步，单市场单品类做到联盟引导 GMV 的个位数百分比即为可观规模。
- 详见 [路线图](./10-roadmap.md) 与 [商业模式](./08-business-model.md) 的单位经济测算。

## 2. 两个市场的结构差异

| 维度 | 🇨🇳 中国 | 🇺🇸 美国 |
|------|---------|---------|
| **主导玩家** | 淘宝/天猫、拼多多、京东、抖音电商 | Amazon、Walmart、Shopify 生态、Google Shopping |
| **CPS / 联盟生态** | 极成熟：淘客、京东联盟、多多进宝、抖音/美团/饿了么 CPS | 成熟：Amazon Associates、CJ、Rakuten、Impact、Awin |
| **支付** | 微信支付 / 支付宝（高度集中） | 卡组织 + PayPal + ACP/Apple Pay；agentic checkout 推进快 |
| **履约预期** | 极快物流、低价、海量 SKU、直播种草 | Prime 次日达、品牌信任、评论文化 |
| **AI 购物现状** | 平台自营 AI（淘宝问问等）为主 | ChatGPT/Perplexity/Gemini/Alexa 多强并存 |
| **合规重点** | PIPL、电商法、广告法、算法推荐规定、数据出境 | FTC 披露规则、AI 披露、CCPA / 州隐私法 |
| **文化** | 价格敏感、爱比价、社交驱动、优惠玩法复杂 | 品牌与评价驱动、看重退货与服务、隐私意识强 |

> **战略含义**：核心需求大脑两市场同构，但**供给接入、履约、合规、文化表达必须本地化**。详见 [系统架构 §双区域](./04-system-architecture.md) 与 [合规](./09-trust-safety-compliance.md)。

## 3. AI 购物竞争格局（2026）

当下有五类玩家正在争夺"购物的入口"，但都不是 Zeno 的形态：

### 3.1 货架平台的自营 AI —— Amazon Alexa for Shopping / 淘宝问问
- **现状**：Amazon 2026-05 将 Rufus 升级为 **Alexa for Shopping**，2025 服务 3 亿+ 用户；阿里亦在淘宝内嵌 AI 导购。
- **优势**：自有库存、履约、评论数据，店内体验最佳。
- **致命短板**：**利益不中立**——只推自家货架，几乎不可能客观地告诉你"去别处买更好/更便宜"。跨平台比价是它的结构性禁区。
- **Zeno 切口**：做它**不能**做的中立跨平台推荐。

### 3.2 通用 AI 助手 —— ChatGPT / Perplexity / Gemini / Copilot
- **现状**：ChatGPT（9 亿周活，已放弃 Instant Checkout、回退到发现）、Perplexity（强在无偏研究与完成购买）、Gemini（Universal Cart + agentic checkout + 虚拟试穿）。
- **优势**：通用智能强、流量巨大。
- **短板**：购物是**边角功能**，不是产品核心；供给侧靠通用抓取 / 有限 feed，缺乏深度商品理解、稳定履约、归因与（尤其中国的）本地供给；记忆与画像不为购物优化。
- **Zeno 切口**：做**购物专属**的深度——更懂商品、更懂你、更能买到、更本地化。

### 3.3 搜索式 AI 导购 —— Google AI Mode / Shopping Graph
- **优势**：商品图谱大、intent 检索强、虚拟试穿、支付内嵌。
- **短板**：基因仍是**检索 + 货架式结果**，偏"找到商品"而非"理解需求并替你做决定"；广告模式带来与中立性的张力。
- **Zeno 切口**：从"intent 检索"升维到"需求理解 + 替你收窄决策"。

### 3.4 导购 / 返利 / 比价工具 —— 什么值得买 / 返利网 / Honey / Rakuten / Wirecutter
- **优势**：有用户信任（尤其编辑测评类）、有返利钩子。
- **短板**：**没有需求理解**——用户仍要自己逛、自己筛；体验是工具而非对话；个性化弱。
- **Zeno 切口**：把"工具 + 人工测评"升级为"AI 需求理解 + 个性化 + 对话"。

### 3.5 内容种草 —— 抖音 / 小红书 / TikTok Shop / 直播
- **优势**：强内容、强转化、强冲动消费。
- **短板**：**非需求驱动**——是"刷到什么买什么"，不是"我要解决问题"；信息噪声大、易冲动后悔。
- **Zeno 切口**：服务"我有明确/半明确需求、想高效选对"的场景（种草的下游决策环节）。

## 4. 竞品矩阵与 Zeno 的位置

用两个轴定位：**「中立性」**（是否客观跨平台）× **「需求理解深度」**（理解品牌 vs. 理解需求）。

```
   中立 ▲
        │   什么值得买/Wirecutter         ★ Zeno
        │   Honey/Rakuten(返利)        (中立 + 深度需求理解
        │                               + 双边市场 + 履约)
        │   Perplexity ─────────────────────────────
        │            ChatGPT/Gemini(购物功能)
   ─────┼──────────────────────────────────────────▶ 需求理解深度
        │   Google Shopping
        │            淘宝/拼多多/京东(货架搜索)
        │   抖音/小红书(种草)   Amazon Alexa for Shopping
   自营/ │                      淘宝问问(自营AI)
   竞价 ▼
```

**Zeno 独占右上角**：既中立、又深度理解需求、还自带双边市场与履约。其它玩家各占一隅，但都因结构性原因（自营利益 / 边角功能 / 工具属性）无法同时占据这三点。

## 5. 起步品类选择

不铺全品类，选**决策成本高、AI 价值最大**的品类切入。理想品类满足：①参数复杂/选择困难；②客单价适中偏高（佣金可观）；③需求可被语言清晰表达；④购买频次与复购合理。

候选（建议 MVP 选 1–2 个深耕）：
- **消费电子 / 数码配件**：参数密集、最适合 AI 解释取舍；中美通吃。
- **母婴 / 宠物**：需求强、信任敏感、复购高、用户愿意"被建议"。
- **家居 / 小家电**：场景化需求（"小户型""租房""降噪"），AI 收窄价值大。
- **美妆个护**：高频、强个性化（肤质/需求），但真实性与合规要求高。
- **运动户外**：场景驱动（"入门跑鞋""通勤背包"），适合需求式表达。

> 选品逻辑与扩张顺序详见 [路线图 §阶段](./10-roadmap.md)。

## 6. 关键风险与应对

| 风险 | 说明 | 应对 |
|------|------|------|
| **平台封锁供给** | 淘宝/Amazon 收紧联盟接口或封禁 | 多源接入（≥3 联盟）、推进 0 佣金自有供给、商家直连 |
| **巨头自我进化** | Amazon/Google 把中立做得更好 | 它们有结构性利益冲突；Zeno 以专注与中立建立信任品牌 |
| **CPS 佣金被压缩** | 联盟降佣影响阶段一收入 | 单位经济不依赖单一联盟；尽快转向非佣金货币化（[08](./08-business-model.md)） |
| **中立 vs 变现张力** | 货币化诱惑侵蚀中立 | 制度性约束：排序不接受竞价、利益披露（[09](./09-trust-safety-compliance.md)） |
| **AI 幻觉损害信任** | 价格/库存/参数说错 | 真实性优先架构、事实校验、可购买性校验（[05](./05-ai-engine.md)） |
| **双市场分散** | 中美同时做摊薄资源 | 单市场单品类先跑通 PMF，再复制；核心引擎复用 |
| **监管（中国数据出境/算法）** | PIPL、算法推荐规定 | 双区域隔离部署、合规前置（[09](./09-trust-safety-compliance.md)） |

---

## 7. 进入策略小结

1. **单市场单品类先做出 PMF**（建议中国 + 数码 或 美国 + 某高决策品类），用 CPS 验证"需求满足率"。
2. **用需求侧体验建立信任品牌**，积累商品知识图谱与用户记忆这两项复利资产。
3. **流量起来后，用 0 佣金撬动供给侧**，把借来的供给变成自有。
4. **复制到第二市场 / 第二品类**，核心引擎复用，边缘本地化。

## ⚖️ 正反方辩论（Red Team / Blue Team）

> 对本文关键主张做一次对抗性审视——正方力挺、反方进攻，最后给出裁决与"会改变主意的信号"。

**议题：中美双线 + 选定高决策品类切入，判断对吗？**

**🟦 正方（支持）**
- 双市场放大规模与数据优势，核心引擎可复用，边际成本递减。
- CPS 生态成熟使进入成本极低，"中立 + 需求驱动"的空隙真实存在。

**🟥 反方（质疑）**
- 中美同时做会摊薄资源，合规与文化差异巨大，易两头不到岸。
- "没人做的空隙"也可能是因为伪需求或不赚钱才没人做。
- AI 导流 +700% 是极低基数爆发，绝对量未必撑起商业；竞品迭代快，窗口可能很短。

**⚖️ 裁决**
- 先单市场单品类跑出 PMF，再复制；用真实"需求满足率 + 单位经济"证伪"伪需求"假设。
- 第二市场只做小成本信号探测，不过早重投。
- **Kill 信号：** 单市场单品类 6–9 个月跑不出留存 / 复购与正向单位经济。
