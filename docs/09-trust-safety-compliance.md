# 09 · 信任、安全与中美合规

> 关联文档：[商业模式](./08-business-model.md) · [AI 引擎](./05-ai-engine.md) · [数据模型](./07-data-model.md) · [系统架构](./04-system-architecture.md)

对 Zeno 而言，**信任不是合规副产品，而是核心资产与商业模式本身**。本文定义如何在制度、产品、技术三层守住信任，并满足中美监管。

---

## 1. 中立性：从原则到制度

中立不能只是口号，必须**制度化、可验证**：

| 制度 | 落地 |
|------|------|
| **佣金不入排序** | `commission_rate` 仅用于结算，被架构性地排除在排序信号之外（[05 §3](./05-ai-engine.md)、[07](./07-data-model.md)） |
| **广告与推荐隔离** | 赞助位（B5）显著标注、与"为你推荐"物理隔离、占比设硬上限，绝不混入自然结果（[08](./08-business-model.md)） |
| **利益披露** | 含返佣链接明确标注；"返佣高低不影响排序"的承诺对用户可见可查（[03 §6](./03-ux-conversational-commerce.md)） |
| **可解释 + 可溯源** | 每个推荐的理由对应用户约束、可回链数据来源（[05 §6](./05-ai-engine.md)） |
| **敢于反向推荐** | 允许并鼓励"去别处更便宜""这个不适合你"——中立的最强信号（[03 §3](./03-ux-conversational-commerce.md)） |
| **可审计** | 排序信号、披露、广告占比留痕，支持内部/外部审计 |

> 一句话：**让"对用户最好"与"对 Zeno 最好"尽量是同一件事**；二者冲突时，制度保证选用户。

## 2. AI 真实性与安全

承接 [05 §6](./05-ai-engine.md)，从信任与安全角度补充：

| 风险 | 防护 |
|------|------|
| **幻觉**（价格/参数/库存说错） | 事实取自工具非模型；展示前可购买性校验；不确定显性标注 |
| **夸大/误导推荐** | 输出合规守卫（见 §3/§4 用语红线）；口碑带样本与出处 |
| **Prompt 注入** | 商品文案/评论等不可信内容做隔离与净化，不让其劫持系统指令 |
| **有害/不当推荐** | 安全策略过滤违禁、受限、对特定人群有害的商品（如对儿童/医疗场景） |
| **操纵性话术** | 禁止制造虚假紧迫感（"仅剩 1 件"类）除非真实且可核验 |
| **偏见/公平** | 评测集覆盖人群与场景多样性，防止系统性偏向某类供给（[05 §7](./05-ai-engine.md)） |

## 3. 中国合规（🇨🇳）

| 法规 / 规定 | 对 Zeno 的要求 | 落地 |
|-------------|----------------|------|
| **《个人信息保护法》(PIPL)** | 最小必要、明确同意、可查改删、**数据出境**严格限制 | 个人数据**境内驻留不出境**；记忆需明确同意、可删；双区域隔离（[04 §5](./04-system-architecture.md)、[07 §3](./07-data-model.md)） |
| **《电子商务法》** | 平台对经营者资质、商品信息、消费者权益尽责 | 商家资质审核、商品真实性、纠纷与售后通道（[06 §4](./06-supply-side.md)） |
| **《广告法》** | 禁绝对化用语（"最/第一/国家级"）、禁虚假/夸大功效；广告需可识别 | 输出守卫过滤违规用语；赞助位标"广告"；功效类克制（§1、[05 §6](./05-ai-engine.md)） |
| **《互联网信息服务算法推荐管理规定》** | 算法透明、可解释、提供关闭个性化选项、不滥用 | 可解释推荐、一键关个性化、算法机制存档备查 |
| **价格法 / 明码标价** | 价格真实、优惠规则透明 | 到手价透明展示，禁误导性原价（[03 §7](./03-ux-conversational-commerce.md)） |
| **数据安全 / 网络安全相关** | 分级保护、安全评估 | 境内云、安全基线、必要时安全评估 |

## 4. 美国合规（🇺🇸）

| 法规 / 规则 | 对 Zeno 的要求 | 落地 |
|-------------|----------------|------|
| **FTC 联盟披露**（Endorsement Guides / 16 CFR 255） | 含佣金关系必须**清晰显著披露** | 返佣链接显著标注；"含返佣"贯穿展示（§1、[03 §6](./03-ux-conversational-commerce.md)） |
| **FTC Act §5（不公平/欺骗）** | 不得欺骗性宣传、虚假稀缺、误导价格 | 真实性优先；禁虚假紧迫感；价格透明 |
| **AI 披露趋势** | 显著告知用户在与 AI 交互、AI 生成内容 | 明确标识 Zeno 为 AI；推荐为 AI 生成、可能有误，提供核验 |
| **CCPA/CPRA 及各州隐私法** | 知情、选择退出"出售/共享"、可访问/删除 | 隐私中心：查看/导出/删除记忆、opt-out；个人数据美国驻留（[07 §3](./07-data-model.md)） |
| **儿童隐私（COPPA）** | 不面向 13 岁以下收集数据 | 年龄门槛与策略 |
| **可访问性（ADA 相关）** | 合理无障碍 | 满足基本可访问性（[02 NFR](./02-product-requirements.md)） |

## 5. 内容与供给安全（中美共用 + 本地分支）

| 主题 | 策略 |
|------|------|
| **假货/侵权** | 富化+质检识别高风险，商家资质与信任分，投诉下架（[06 §4](./06-supply-side.md)） |
| **违禁/受限品** | 按各市场清单过滤（药品、武器、成人、受管制品等），本地化规则 |
| **虚假评论/刷单** | 检测异常评论与刷单，富化时不向用户传递可疑口碑 |
| **错价/薅羊毛价** | 异常价过滤，避免误导与履约纠纷 |
| **未成年人保护** | 内容与品类的年龄适配 |

## 6. 隐私与数据治理（贯穿）

- **区域驻留**：个人数据 CN/US 各自境内不跨境（PIPL 出境 + 美国州法），仅"非个人"资产（本体/模型/评测集）跨区复用（[04 §5](./04-system-architecture.md)）。
- **用户控制**：记忆与画像可**查看 / 编辑 / 导出 / 删除**；个性化基于明确同意、可一键关闭（[07 §3](./07-data-model.md)）。
- **最小化**：只采集为完成推荐与履约所必需的数据。
- **脱敏**：归因等链路对用户标识哈希/脱敏（[07 §5](./07-data-model.md)）。
- **留存**：明确留存期与到期清理。

## 7. 信任治理机制

| 机制 | 说明 |
|------|------|
| **中立审计** | 定期审计排序是否被佣金/广告污染、披露是否到位、广告占比是否超限 |
| **可解释抽检** | 抽检推荐理由是否忠实于数据（防"事后编理由"） |
| **用户申诉** | 用户可质疑推荐/价格/披露，形成反馈闭环（[11](./11-metrics.md)） |
| **红队** | 定期对 AI 做对抗测试（注入、诱导、有害推荐） |
| **透明报告** | 适时公开中立性与安全治理报告，建立外部信任 |

---

## 8. 合规与产品的协同原则

1. **合规前置，不是补丁**：双区域、披露、用语守卫在架构与流程里内建。
2. **信任即增长**：中立与透明降低 CAC、提升复购，是商业的一部分而非成本（[08](./08-business-model.md)）。
3. **冲突时选用户**：任何"短期多赚但伤信任"的选择，默认不做。
4. **本地化分支、核心同构**：合规边缘本地化，信任原则两市场一致。

> **底线**：Zeno 可以少赚钱，但不能让用户怀疑"它到底站在谁那边"。
