# 10 · 路线图与里程碑

> 关联文档：[PRD](./02-product-requirements.md) · [供给侧](./06-supply-side.md) · [度量](./11-metrics.md) · [愿景](./00-product-vision.md)

路线图遵循一条主线：**先用 CPS 在单市场单品类做出 PMF（需求满足率），再把借来的供给换成自有，最后复制到双市场并平台化。** 时间为相对节奏（非承诺日期），以"门槛指标"驱动进入下一阶段。

---

## 阶段总览

```
P0 MVP          P1 深化            P2 自有供给+双市场      P3 平台化
单市场·单品类    多品类·记忆·履约    0佣金入驻·第二市场       开放·生态·规模
CPS 跑通体验     做厚护城河          沉淀自有供给+ACP直购     双边市场成熟
   │               │                    │                      │
   ▼               ▼                    ▼                      ▼
验证"AI能           验证"复购与          验证"0佣金能换           验证"网络效应
找到对的货"         个性化复利"          来更优自有供给"          与规模经济"
```

---

## P0 · MVP（验证核心假设）

**目标**：证明"用户描述需求 → AI 能从全网找到对的商品并被采纳"。

**范围（一句话）**：**一个品类（如数码）× 一个市场（如中国 CPS）× 一个杀手体验**（描述需求 → AI 收窄 → 解释推荐 → 一键履约）+ 真实价格/库存 + 利益披露。

**必做（Must / P0）**
- 需求理解 + 最小澄清（[05 §1](./05-ai-engine.md)）
- 混合检索 + 多目标排序 → 2–4 候选 + 可解释（[05 §2/§3](./05-ai-engine.md)）
- 结构化对比（[03](./03-ux-conversational-commerce.md)）
- ≥1 中国 + ≥1 美国联盟其一打通：标准化 + 富化 + 去重 + 到手价（[06 §2](./06-supply-side.md)）
- 跳转式履约 + 归因对账闭环（[06 §2.3](./06-supply-side.md)）
- 真实性保障 + 利益披露（[05 §6](./05-ai-engine.md)、[09](./09-trust-safety-compliance.md)）
- 评测闭环：黄金需求集 + 在线需求满足率埋点（[11](./11-metrics.md)）

**不做**：0 佣金入驻、站内支付、语音、跨市场、全品类。

**门槛指标（进入 P1，假设值待校准）**
- 需求满足率 ≥ 目标线；推荐采纳率显著高于关键词搜索基线。
- 幻觉率（价格/参数）低于硬上限。
- 单次对话 AI 成本可控、单位经济为正或路径清晰。

**主要风险**：富化质量不足→匹配差（缓解：聚焦单品类把富化做透）；联盟接口变动（缓解：归因对账稳健、预留多源）。

---

## P1 · 深化（做厚护城河）

**目标**：从"能找到对的货"升级到"越用越懂你"，验证复购与个性化复利。

**范围**
- **多品类扩展**（在起步市场内，按 [01 §5](./01-market-analysis.md) 选品逻辑）。
- **多联盟接入** + 同款跨平台比价成熟（[06](./06-supply-side.md)）。
- **个性化记忆**：偏好、拒绝信号、已购去重；心愿 + 降价/替代提醒（[07 §3](./07-data-model.md)）。
- **图片多模态**（找同款/找类似）（[03 §5](./03-ux-conversational-commerce.md)）。
- **到手价玩法**完善（券/返利叠加，本地化）。
- 需求侧**会员（Plus）**试水（[08](./08-business-model.md)）。

**门槛指标（进入 P2）**
- 复购率 / 留存达标；记忆带来的采纳率与 LTV 提升可量化。
- 多品类需求满足率稳定；供给覆盖与质量达标。

**主要风险**：品类扩张摊薄质量（缓解：每品类过富化与评测门槛再上线）。

---

## P2 · 自有供给 + 双市场（换护城河 + 复制）

**目标**：把借来的供给换成**自有且更优**，并复制到第二市场。

**范围**
- **0 佣金商家入驻** + 商品录入/导入 + **经营看板** + 增值服务（[06 §3](./06-supply-side.md)、[08](./08-business-model.md)）。
- **供给治理**成熟：信任分、质量分级、假货/违禁处置（[06 §4](./06-supply-side.md)、[09](./09-trust-safety-compliance.md)）。
- **站内 agentic checkout（ACP）直购** + 支付分成（B4）（[08](./08-business-model.md)）。
- **第二市场上线**：核心引擎复用，供给/履约/合规本地化（[04 §5](./04-system-architecture.md)）。
- 语音、售后闭环（Could）。

**门槛指标（进入 P3）**
- 自有（0 佣金）供给占比与独家/更优价比例达标。
- 非佣金收入线（履约/工具/会员/ACP）ARPU 跑通。
- 第二市场达到第一市场 PMF 同等的需求满足率。

**主要风险**：双市场资源分散（缓解：第一市场 PMF 稳固后再大力投入第二市场）；0 佣金冷启动（缓解：用需求侧流量换供给，种子品类聚焦，[06 §3.3](./06-supply-side.md)）。

---

## P3 · 平台化（规模与生态）

**目标**：让双边飞轮与网络效应进入规模经济，建立行业级地位。

**范围（方向性）**
- 商家生态开放（更多工具、API、第三方服务）。
- 知识图谱与需求理解能力对外赋能（受控）。
- 更深履约与会员体系；跨市场协同（如跨境）。
- 中立治理透明化（透明报告、外部审计，[09 §7](./09-trust-safety-compliance.md)）。

**门槛指标**：双边网络效应可量化（供给增长带动需求满足率提升的正反馈）；单位经济随规模改善。

|------|--------|-----------|-----------|----------|------|
| **P0** | 单市场单品类 CPS 跑通 | 理解+收窄+解释+履约 | ≥1 联盟标准化+归因 | CPS 佣金 | 需求满足率/采纳率达标 |

## 贯穿各阶段的不变量

无论哪个阶段，这几条始终不破（来自 [愿景 §7](./00-product-vision.md)、[合规](./09-trust-safety-compliance.md)）：

1. **佣金/广告不污染排序**——中立是底线。
2. **真实可买优先于流畅**——不拿幻觉换体验。
3. **轻资产**——不自建库存物流（前三阶段）。
4. **隐私与合规前置**——双区域、可查改删、本地化合规。
5. **以需求满足率为北极星**——不被 GMV/SKU 数等虚荣指标带偏（[11](./11-metrics.md)）。

## ⚖️ 正反方辩论（Red Team / Blue Team）

> 对本文关键主张做一次对抗性审视——正方力挺、反方进攻，最后给出裁决与"会改变主意的信号"。

**议题：P0→P3 线性阶段 + 门槛指标驱动 + 单点先行，节奏对吗？**

**🟦 正方（支持）**
- 单点 PMF 再复制，最大限度降低烧钱试错风险。
- 门槛指标防止盲目扩张、强迫每阶段验证真实价值。

**🟥 反方（质疑）**
- 阶段过于线性，市场窗口短可能等不及；竞品并行推进会抢身位。
- 门槛指标可能被刷或定义不当，导致过早 / 过晚推进。
- "先中国数码 or 美国某品类"本身是高风险一次性下注，押错满盘皆输。

**⚖️ 裁决**
- 重注单点，但用极小成本并行探测第二市场 / 第二品类的信号，不把鸡蛋全压一处。
- 门槛指标一律配反指标防刷（见 [11](./11-metrics.md)）；选品用数据快速试错而非豪赌。
- **Kill 信号：** P0 长期达不到门槛，且更换品类 / 市场后仍无改善。
