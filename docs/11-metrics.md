# 11 · 度量体系与北极星

> 关联文档：[愿景](./00-product-vision.md) · [AI 引擎](./05-ai-engine.md) · [商业模式](./08-business-model.md) · [路线图](./10-roadmap.md)

度量的目的不是堆指标，而是**回答一个问题：Zeno 是否真的高效地让对的需求遇见了对的供给，且用户事后不后悔？** 指标体系据此组织，并刻意设置**反指标**防止被刷。

---

## 1. 北极星指标（North Star）

> **被满足的真实需求量（Fulfilled Needs）**
> ＝ 用户表达需求 → Zeno 给出推荐 → 用户**采纳并完成购买** → 且**事后未后悔**（无退货/差评/反悔）的闭环数量。

**为什么是它**：它同时约束三件事——
- 匹配**准不准**（需求侧）、
- 有没有**合适的货**（供给侧）、
- 用户**是否真的满意**（信任）。

任何只优化其一（如只刷成交）都无法抬高它，因为"事后未后悔"这一项把短期操纵挡在门外。

**质量加权版**：`Fulfilled Needs × 满意度`，避免把"勉强成交"算成胜利。

---

## 2. 需求侧指标

| 指标 | 定义 | 意义 |
|------|------|------|
| **需求满足率（Need-Fit Rate）** | 用户认可"这就是我要的"的会话占比（在线信号 + 抽样人工/LLM 评审） | 核心质量，直接喂回 AI 评测（[05 §7](./05-ai-engine.md)） |
| **推荐采纳率** | 用户选择了 Zeno 推荐候选的比例 | AI 是否真比用户自己挑得好 |
| **决策时长 / 轮次** | 从表达需求到决定的耗时与对话轮数 | "高效"的直接度量（越低越好，但不能以牺牲满足率换） |
| **澄清效率** | 平均澄清轮次、澄清后满足率提升 | 澄清是否"问在点上"（[05 §1](./05-ai-engine.md)） |
| **购买转化率** | 决定 → 完成购买 | 履约顺畅度（[06](./06-supply-side.md)） |
| **事后满意 / 后悔率** | 退货、差评、反悔、申诉 | 信任的滞后真相，北极星的关键项 |
| **复购 / 留存** | 周/月活跃、复购间隔 | 个性化复利是否成立（[07 §3](./07-data-model.md)） |

---

## 3. 供给侧指标

| 指标 | 定义 | 意义 |
|------|------|------|
| **需求覆盖率** | 有多少比例的真实需求"有合适的货可推" | 供给是否够用（比"SKU 总数"有意义得多） |
| **供给质量分** | 加权 quality_score、退货率、商家信任分 | "Zeno 推荐=可信"的保障（[06 §4](./06-supply-side.md)） |
| **同款比价覆盖** | 能跨平台给出最低到手价的商品占比 | 比价能力（用户核心价值） |
| **富化质量** | 属性抽取准确率、场景/人群标注质量 | 匹配上限（[05 §5](./05-ai-engine.md)） |
| **价格/库存新鲜度** | 失效率、可购买性校验通过率 | 真实可买（[05 §6](./05-ai-engine.md)） |
| **自有供给占比（P2+）** | 0 佣金入驻供给 / 总供给 | 护城河深度（[06 §3](./06-supply-side.md)、[10](./10-roadmap.md)） |
| **CPS 引导 GMV / 佣金** | 经 Zeno 成交的额与佣金 | 阶段一收入（[08](./08-business-model.md)） |

---

## 4. AI 质量指标（喂回引擎）

| 指标 | 定义 |
|------|------|
| **硬约束零违反率** | 推荐绝不出现被排除/超预算项的比例 |
| **解释忠实度** | 推荐理由真实对应数据（非事后编造）的抽检通过率 |
| **幻觉率** | 价格/参数/库存错误率（硬上限） |
| **候选多样性** | 候选间是否有意义地不同（非近似重复） |
| **黄金集回归分** | 在离线黄金需求集上的契合度（[05 §7](./05-ai-engine.md)） |
| **澄清信息增益** | 澄清问题的平均信息增益 |

> 闭环：线上 badcase → 回流黄金集 → 回归 → 迭代富化/排序/提示（[05 §7](./05-ai-engine.md)）。

## 5. 商业与增长指标

| 指标 | 说明 |
|------|------|
| **单位经济** | 单次撮合贡献毛利、AI 成本、缓存命中率（[08 §3](./08-business-model.md)） |
| **LTV / CAC** | 目标 ≥ 3（待验证）；中立信任降 CAC，记忆提 LTV |
| **非佣金 ARPU（P2+）** | 履约/工具/会员/ACP 收入（[08 §2](./08-business-model.md)） |
| **0 佣金入驻渗透** | 商家入驻数/活跃/留存（[06 §3](./06-supply-side.md)） |
| **双边飞轮系数** | 供给增长 → 需求满足率提升 的正反馈强度（[08 §4](./08-business-model.md)） |

---

## 6. 信任指标

| 指标 | 说明 |
|------|------|
| **中立审计通过率** | 排序未被佣金/广告污染、披露到位、广告占比不超限（[09 §7](./09-trust-safety-compliance.md)） |
| **披露可见度** | 用户对"含返佣/这是广告"的感知与理解 |
| **反向推荐占比** | 敢于"劝退/指向他处"的比例（健康的中立信号，不应为 0） |
| **申诉解决率/时长** | 用户质疑推荐/价格/披露的处理 |
| **后悔率（与北极星共享）** | 信任的滞后真相 |

---

## 7. 反指标（Counter-Metrics，防止被刷）

每个增长指标都配一个"刹车"，防止以伤害用户为代价做数：

| 若只看… | 会被这样刷… | 配套反指标 |
|---------|-------------|-----------|
| 成交 GMV | 推高佣金品/冲动购买 | **后悔率 / 退货率 / 满意度** |
| 推荐采纳率 | 只推安全平庸的大众款 | **候选多样性 / 需求满足率（含长尾难需求）** |
| 决策时长↓ | 草率收窄、少给信息 | **事后满意率 / 解释忠实度** |
| SKU/商品总数 | 堆量不堆质 | **需求覆盖率 / 供给质量分** |
| CPS 收入 | 佣金污染排序 | **中立审计通过率 / 佣金-排序相关性≈0** |
| 会话轮次/时长 | 制造黏着但低效 | **每轮信息增益 / 决策效率** |

> **设计原则**：北极星（被满足的真实需求量，含"未后悔"）本身就是最强反操纵约束；其余反指标是第二道防线。

---

## 8. 度量治理

- **分市场 / 分品类拆解**：所有核心指标按 CN/US、按品类下钻（避免平均值掩盖问题）。
- **在线 + 离线双轨**：在线行为信号 + 离线人工/LLM 评审，互相校准（防裁判偏差，[05 §7](./05-ai-engine.md)）。
- **实验平台**：模型、排序权重、澄清策略、UI 均可 A/B，以北极星与反指标共同判定胜负。
- **隐私合规**：度量埋点遵循数据驻留与脱敏（[09](./09-trust-safety-compliance.md)）。
- **阶段门槛**：各阶段进入下一阶段的"门槛指标"见 [路线图](./10-roadmap.md)。

> **一句话**：我们只认一个胜利——**用户得到了真正适合自己的东西，并且不后悔**。其余指标都是为这个胜利服务，或为防止伪造这个胜利而存在。
