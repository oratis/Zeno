# 06 · 供给侧设计：CPS 接入 + 0 佣金入驻

> 关联文档：[数据模型](./07-data-model.md) · [商业模式](./08-business-model.md) · [系统架构](./04-system-architecture.md) · [市场分析](./01-market-analysis.md)

供给侧的目标是**供给端高效**：让"对的商品"以最低摩擦被"对的需求"找到。策略分两阶段——**先借（CPS），后建（0 佣金）**。

---

## 1. 两阶段战略

```
阶段一：CPS 接入（轻资产冷启动）          阶段二：0 佣金入驻（沉淀自有供给）
─────────────────────────────            ─────────────────────────────────
接入全网已存在的高质量商品库      ──流量起来──▶  用 0 佣金把商家直接引入 Zeno
零库存、第一天就有海量供给+真实收入            获得独家商品/更优价格/更深履约
解决"鸡生蛋"冷启动                            建立护城河，非佣金方式变现
```

**为什么是这个顺序**：先 CPS，让产品在"没有自有供给"时也能给用户**完整体验**（破冷启动）；当需求侧跑通、有真实流量后，0 佣金才成为对商家极具吸引力的 wedge，把供给从"借来的"变成"自己的"。详见 [愿景 §4.2](./00-product-vision.md)。

## 2. 阶段一：CPS 联盟接入

### 2.1 接入哪些联盟

| 市场 | 联盟 | 跟单标识 | 备注 |
|------|------|----------|------|
| 🇨🇳 | **淘宝联盟（淘客）** | PID（含 pid/adzone） | 生态最成熟、SKU 最全 |
| 🇨🇳 | **京东联盟** | SubUnionId | 高客单、低退货、跨店结算；对开发者友好 |
| 🇨🇳 | **多多进宝（拼多多）** | custom_parameters.uid | 对个人/开发者最友好、价格敏感人群 |
| 🇨🇳 | **抖音 / 美团 / 饿了么 CPS** | 各自参数 | 内容电商 / 本地生活，按品类拓展 |
| 🇺🇸 | **Amazon Associates** | tag / SubID | 覆盖最广，佣金 1–10%（按类目） |
| 🇺🇸 | **CJ Affiliate** | PID + SID | 品牌多，佣金均值约 15–25% |
| 🇺🇸 | **Rakuten / Impact / Awin** | 各自 SubID | 高端品牌、creator 友好（Impact 20–40%） |

> **多源接入是抗风险设计**：任一平台收紧/降佣，不致命。优先打通"≥1 中国 + ≥1 美国"跑通闭环（[路线图](./10-roadmap.md)）。

### 2.2 接入与标准化管道

```
联盟 API/Feed
   │  (商品/类目/价格/券/佣金率/详情)
   ▼
① 接入 Ingest        全量 + 增量；限频与配额管理；多源并行
   ▼
② 清洗 Clean         去 HTML/噪声、单位/币种归一、缺失补全
   ▼
③ 标准化 Normalize   映射到「统一商品模型」(类目/属性字段对齐)  ← [07]
   ▼
④ 富化 Enrich        LLM 抽属性/场景/人群、口碑提炼、生成语义向量  ← [05 §5]
   ▼
⑤ 去重 Dedup         跨平台同款合并(同一商品多个购买渠道→1个实体, N个offer)
   ▼
⑥ 质检 QA            违禁/假货/异常价/低评分/失效过滤
   ▼
⑦ 索引 Index         向量 + 结构化 + 全文 + 知识图谱
   ▼
近实时同步：价格/库存/券/失效 高频更新(流式) → 保证"真实可买"
```

**关键设计**
- **同款合并（核心）**：同一商品在淘宝/京东/拼多多/Amazon 可能各有一条记录。Zeno 把它们合并为**一个商品实体 + 多个购买 offer**，从而能**跨平台比"到手价"**——这是货架平台做不到、也是用户最想要的。
- **到手价计算**：聚合可叠加的券/返利/满减（中国）与 coupon/cashback（美国），算出真实到手价用于排序与展示。
- **富化是上限**：检索/匹配质量取决于富化质量，详见 [05 §5](./05-ai-engine.md)、[07](./07-data-model.md)。

### 2.3 归因与对账

- **链路生成**：履约时为选定 offer 生成带正确跟单标识（PID/SubID/uid/tag）的购买链接；未来站内 ACP 直购（[08](./08-business-model.md)）。
- **归因回收**：从各联盟拉取成交/佣金报表，与 Zeno 的引导事件匹配（订单号/点击 ID/时间窗）。
- **对账**：处理跨平台口径差异、退款冲正、结算周期；产出可信的收入与单位经济数据（[08](./08-business-model.md)、[11](./11-metrics.md)）。
- **归因与排序解耦**：归因数据**不回流到排序**作为加权，守住中立（[09](./09-trust-safety-compliance.md)）。

### 2.4 阶段一的供给质量控制

即使是借来的供给，质量也由 Zeno 背书。上架前/排序中过滤与降权：
- 失效/无货/价格异常（薅羊毛价、错价）→ 不展示或标注。
- 违禁/受限/假货高风险 → 过滤（中美各按本地合规，[09](./09-trust-safety-compliance.md)）。
- 低评分/高退货/差商家信任分 → 降权。
- 虚假宣传/夸大功效文案 → 富化时清洗，不向用户传递。

## 3. 阶段二：0 佣金商家入驻

### 3.1 为什么是 0 佣金

- **对商家**：货架平台用高佣金 + 高买量成本挤压利润。Zeno 0 成交佣金 + 按需求精准匹配 = **更低获客成本、更高质量流量**，这是强力 wedge。
- **对 Zeno**：0 佣金不是 0 收入。靠**非佣金**方式变现（履约、营销工具、数据洞察、会员、支付分成），使平台收入**不与商家 GMV 直接对立**，从而**保住中立**——这正是货架广告模式的死结，也是 Zeno 的结构性优势。详见 [商业模式](./08-business-model.md)。

> 核心理念：**不靠"对商家抽税"赚钱，靠"帮买卖双方更高效地相遇"赚钱。**

### 3.2 商家侧能力（Merchant Platform）

| 能力 | 说明 |
|------|------|
| **自助入驻** | 资质审核、店铺创建、信任分初始化 |
| **商品录入/导入** | 手动 / 批量 / 从现有店铺导入；自动走标准化+富化管道 |
| **需求化商品描述** | 引导商家用"适合谁/什么场景"描述商品，映射到知识图谱，**提升被匹配概率**（供给侧也要"说需求语言"） |
| **经营看板** | 曝光 / 被匹配 / 转化 / **被拒原因**（哪类需求没选你、差在哪），指导商家改进供给 |
| **增值服务（付费）** | 履约（仓配/客服）、营销工具、数据洞察、会员权益——非佣金变现（[08](./08-business-model.md)） |

### 3.3 0 佣金的冷启动

经典双边鸡生蛋。破法：
1. **先有需求**：阶段一用 CPS 撑起完整体验，先把**需求侧流量**做起来。
2. **拿流量换供给**：对商家展示"这里有 N 个在找你这类商品的真实需求"，0 佣金 + 精准流量是入驻理由。
3. **种子品类聚焦**：在 1–2 个起步品类把"需求-供给"打透，做出标杆，再横向扩。
4. **独家激励**：早期入驻商家给更多曝光/数据/工具权益，换独家或更优价格。

## 4. 供给治理（跨阶段）

| 机制 | 说明 |
|------|------|
| **商家/商品信任分** | 综合评分、退货率、纠纷、虚假宣传记录；影响排序权重与展示 |
| **假货/违禁识别** | 规则 + 模型 + 人工，中美各按本地合规处置（[09](./09-trust-safety-compliance.md)） |
| **价格/库存新鲜度** | 近实时同步 + 展示前可购买性校验（[05 §6](./05-ai-engine.md)） |
| **质量分级** | 优质供给加权、劣质降权或下架；保护"Zeno 推荐 = 可信"的品牌资产 |
| **反操纵** | 防刷单、防虚假评论、防关键词堆砌；富化与质检环节拦截 |

## 5. 供给飞轮

```
        更多真实需求(需求侧体验好)
              │
              ▼
        更多被引导的成交 (GMV / CPS 收入)
              │
              ▼
   更多商家愿意 0 佣金入驻 (精准流量诱人)
              │
              ▼
   更优质 / 更独家 / 更低价的供给
              │
              ▼
        更精准的匹配 + 更好到手价
              │
              └──────────▶ 更多真实需求 (回到顶部)
```

- 阶段一用 **CPS** 让飞轮**带着借来的供给先转起来**；
- 阶段二用 **0 佣金** 把飞轮的"供给"换成**自有且更优**的，越转护城河越深。
- 双边网络效应一旦成立，后来者难追（[愿景 §6](./00-product-vision.md)）。

## 6. 供给侧里程碑（摘要）

| 阶段 | 供给侧目标 |
|------|-----------|
| **P0 / MVP** | 打通 ≥1 中国 + ≥1 美国联盟；起步品类标准化+富化+去重+到手价；归因对账闭环 |
| **P1** | 多联盟、多品类；同款跨平台比价成熟；质量分级与近实时同步稳定 |
| **P2** | 0 佣金商家入驻 + 经营看板 + 增值服务；自有供给占比提升；站内 ACP 直购 |
